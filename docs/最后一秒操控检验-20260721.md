# "最后一秒"操控检验（2026-07-21）

## 检验框架

针对"平台是否在封盘前最后一秒操控数据（如强行达到某指标）"，建立三个可复用检验：

| 检验 | 原理 | 判定 |
|---|---|---|
| A 外推残差 | 用封盘前 4~20s 的资金曲线线性外推末帧，残差=实际−预测 | 残差分布畸形=有人为注入/抽离 |
| B 残差方向×结果（核心） | 庄/闲池分别外推，检验尾盘残差是否系统性偏向最终获胜方 | 显著偏胜方=内幕资金/操控；显著偏负方=杀尾盘 |
| C 封盘帧间隙 | 末帧与 countdownEndTime 的时间差分布 | 异常拉长/双峰=平台 Hold 数据做决策 |

## 数据

- 全桌型 13,099 局（rounds 有 ts_bet_end 且 ≥6 帧 110 快照），A/B 有效样本 12,163 局
- 全部原始帧存于 events_raw，检验可复现

## 结果

### A 总池相对残差
中位 +24.3%，均值 +188.9%（巨额局拉高）；60.6% 的局残差 >+10%，20.4% <-10%。
分布正偏——符合"尾盘自然加速"（凸增长，线性外推系统性低估），**无畸形特征**。

### B 胜方残差 − 负方残差（归一化）
- 均值 +0.057，95%CI [-0.348, +0.463]，跨 0；
- 残差偏胜方的局占 **48.8%**（二项 95%CI [47.9%, 49.7%]），名义 p≈0.008 低于 50% 基准，
  方向反而是"尾盘钱略偏向**输**的一方"，但效应仅 1.2pp，且不过多重检验校正。
- **结论：12,163 局全量，最后一秒的资金没有系统性流向获胜方——无内幕资金特征。**

### C 封盘帧间隙
中位 -1.58s，p99 -0.92s（末帧平均在名义封盘时刻后 1.6 秒到达，即实际下注窗口比倒计时长约 1.5s）。
间隙 >2s 占 0%，无拉长、无双峰——**平台没有 Hold 帧行为，110 流覆盖到实际封盘**。

## 结论

1. 全量牌局的"最后一秒"**未发现数据操控嫌疑**：残差分布自然、无方向性偏向、帧推送无截留；
2. 唯一名义显著的 B 检验方向（尾盘钱略偏输方）效应 1.2pp，与此前"最后一跳大钱胜率 ~27%"同向但弱，
  合并解读为"尾盘从众资金略易被收割"的弱线索，非平台改数证据；
3. 平台若操控牌局结果，其决策依据**不是池级资金数据**（我们全量可观测面上无痕迹），
  操控点只能在牌靴/发牌物理层，这与"平台干净的过分"的既有研判一致。

## 复现与口径

- 外推窗口 [-20s,-4s]，线性最小二乘，末帧为预测点；B 检验剔除和局（T），B6 计入庄胜；
- countdownEndTime 为名义封盘时刻，实际帧可延后 ~1.5s（服务器节奏），检验已覆盖该窗口；
- 本检验只覆盖池级聚合数据；逐笔级数据协议不提供，无法检验。

## 补充检验 D：开牌时长异常（2026-07-21 晚）

针对"开牌开到一半等待=在等变牌"的观察：以 107 结果帧到达时刻 − ts_bet_end 度量开牌时长（16,969 局）。

- 时长分布：中位 12.6s，p90 21.3s，p99 37.4s，max 118.8s；
- 重钱方死亡率：正常 49.9% / 偏长 49.9% / 异常长 49.9% / 极端长(≥p99, n=170) 52.4% [44.9,59.7]；
- 仅强偏向局（|imb|>30%）：各层同样 ≈50%。

**结论：开牌时长与结果方向无相关性**——"开牌等待"要么是自然流程波动，
要么平台在等待期间的决策依据不是池级资金（点杀个体/其他信息），聚合层面同样无痕。
