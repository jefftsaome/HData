# HData 打包说明（pyd / so）

将 hdata 编译为二进制扩展模块，对外只暴露 `hdata.client` 的公开接口，隐藏协议与加密实现。

## 环境要求

| 项 | 要求 |
|---|---|
| Python | 3.13.x（与目标运行环境**同一大版本**，.pyd 与解释器强绑定） |
| Cython | >= 3.0（`pip install cython setuptools wheel`） |
| C 编译器 | Windows: **gcc (MinGW)**（scoop 安装即可，`scoop install gcc`）或 MSVC Build Tools<br>Linux: `gcc`<br>macOS: Xcode CLT（`clang`） |

> ✅ 本机已实测：scoop gcc 15.2.0 + `HDATA_COMPILER=mingw32` 可完整编译 39 个模块，
> 产物已在无源码目录验证可正常导入、签名完整、协议编解码往返正确。

## 打包步骤

```bash
# 1. 安装构建依赖
uv add cython    # 或 pip install cython setuptools wheel

# 2. 编译（产物在 dist_pyd/）
#    Windows 用 gcc（MinGW）时：
$env:PATH = "G:\Applications\Scoop\apps\gcc\current\bin;$env:PATH"   # 或你的 gcc 路径
$env:HDATA_COMPILER = "mingw32"
python scripts/build_pyd.py

#    Linux/macOS（或有 MSVC 的 Windows）直接：
python scripts/build_pyd.py
```

产物结构：

```
dist_pyd/
└── hdata/
    ├── __init__.cp313-win_amd64.pyd
    ├── client.cp313-win_amd64.pyd
    ├── auth/*.pyd
    ├── protocol/*.pyd
    └── ...
```

产物中**不含任何 .py 源码**。

## 分发与使用

1. 把 `dist_pyd/hdata` 目录拷贝给调用方（或打成 wheel/zip）。
2. 调用方将其放在 `sys.path` 可达处，并安装运行时依赖：
   `aiohttp websockets curl-cffi pycryptodome cryptography`
3. 使用方式与源码版完全一致：

```python
from hdata.client import GameClient   # 接口契约见 docs/对外接口文档.md
```

## 进一步隐藏实现（可选加固）

- `scripts/build_pyd.py` 已开启 `embedsignature`（保留签名供 `help()` 查阅）；
  如需连签名也隐藏，删除该 directive。
- 可在 `compiler_directives` 中加 `"annotation_typing": False` 减少类型信息泄露。
- 对外只承诺 `hdata.client` 的稳定签名；其余模块视为私有，后续可自由重构。

## 常见问题

- **ImportError: DLL load failed** — 目标机器 Python 大版本不一致，需用同版本重新编译。
- **Linux 出 .so**：同一脚本直接运行即可，产物后缀自动为 `.cp313-x86_64-linux-gnu.so`。
