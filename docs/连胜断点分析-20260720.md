## 二、方法学教训：三个估计器给出三个答案

同一晚数据，三种取数方式给出 0.588 / 0.442 / 0.492 三个"断率"，根因必须记住：

1. **监控 episode 口径（0.588，虚高）**：只用 streak_rounds 里 broke 的 episode 计算时，
   断率显著偏高。原因是监控侧在 episode 存续期间可能漏记个别"续"局
   （WS 帧丢失、进场时机等），漏掉的永远是"续"不可能是"断"（断会关闭 episode），
   分子分母不对称 → 断率被机械抬高。**教训：episode 表的 round 数不能当完整序列用。**
2. **监控桌路纸口径（0.442，虚低）**：rounds 表的路纸来自"因出了长龙才被选中监控"的桌台，
   选择本身偏爱"连性重"的桌台和时段 → 断率被压低。
   佐证：该样本里连对照位置（长度 1–4）的换边率也只有 0.478，整体偏连。
3. **全厅快照口径（0.492，无偏）**：大厅快照覆盖全部桌台全部牌局，无选择偏差，
   且庄占比自证通过。**这是唯一可用于假设检验的口径。**

另两个已踩过的坑：

- **和局混入分母**：T 局既不算断也不算续，混入分母会把断率稀释约 5–6 个百分点
  （v1 的 0.490 含此偏差的一部分）。所有断率统计必须先剔 T。
- **删失 episode 混入**：censored_disconnect / censored_boot 的 episode 只有"续"没有"断"，
  混入会把断率拉低（v1 全体口径 0.447 即受此影响）。假设检验只用 broke episode；
  但如教训 1，即便只用 broke episode，round 级序列仍不可信，只能用于 episode 级统计。

## 三、已排除假设：连胜方注额/人数下降预示断龙（假信号全记录）

**假设**：长龙 4–10 局内，连胜方注额（或人数）较上一局下降 → 下一局更容易断。

**初算曾得出强信号**（下降组断率 0.726 vs 未降组 0.342），后查明是分析代码两个 bug
叠加制造的假相关，与数据无关：

1. **B6 归类 bug**：庄以 6 点赢的局 result 为 `B6` 而非 `B`，被错误归入闲方取注额。
2. **跨池比较 bug（主因）**：断龙那一局赢的是**反方**。按"当前局获胜方"取注额池时，
   断龙局变成"反方池 vs 上一局连胜方池"的跨池比较；庄池与闲池体量有系统性差异，
   断龙局被机械地几乎全部归成"金额下降"，而断龙局按定义 outcome=broke → 假相关。

**修正口径**：连胜方在整个 episode 内固定（取 `streak_episodes.side`），
每局比较"连胜方自己的池 vs 上一局连胜方自己的池"；剔 T；只用 broke episode。

修正后全部无信号（样本 n≈570–620 对）：

| 分组 | 断率 | 95% CI |
|---|---|---|
| 连胜方金额降 | 0.459~0.614（随口径微调波动） | 均覆盖基准 |
| 连胜方金额未降 | 0.491~0.595 | 均覆盖基准 |
| 人数未降 & 金额降（≤100 万池过滤后） | 0.452 | [0.354, 0.553] |
| 降 0–20% / 20–50% / >50% | 0.44 / 0.47 / 0.44 | 无单调关系 |
| 双方都降 / 仅连胜方降 / 仅反方降 / 都未降 | 0.41 / 0.48 / 0.55 / 0.48 | CI 全重叠 |

**结论：注额、人数、双方金额组合、降幅分层，均不能预测断龙。**
