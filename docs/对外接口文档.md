# HData 对外接口文档

> 面向外部调用方的公共 API 说明。本文档只描述**接口契约**（函数签名 / 参数 / 返回值），
> 内部实现（协议、加密、打码、域名解析）对调用方完全透明。

## 安装

```bash
pip install hdata-xxx.whl        # 或将 dist_pyd 目录加入 sys.path
```

运行时依赖：`aiohttp`、`websockets`、`curl-cffi`、`pycryptodome`、`cryptography`。
**Python 版本必须与编译时一致（3.13.x）**。

## 快速开始

```python
import asyncio
from hdata.client import GameClient

async def main():
    client = GameClient(
        entry_url="https://leyu.com",     # 平台入口种子站
        geepass_token="<geepass 打码 token>",
        jfbym_token="<jfbym 打码 token>",
    )

    # 1) 登录
    session = await client.login("账号", "密码")

    # 2) 拉桌台列表
    tables = await client.get_tables()                # 全部桌台
    baccarat = await client.get_tables(game_type_id=2001)  # 只看经典百家乐

    # 3) 进桌，读数据
    async with await client.enter_table(baccarat[0]["table_id"]) as ts:
        print(ts.snapshot)          # 桌内全量快照
        print(ts.road_flat())       # 珠盘路 "BPTBP..."
        async for event in ts.events():
            print(event)            # 持续牌局事件流

asyncio.run(main())
```

---

## `GameClient`

```python
GameClient(entry_url, geepass_token="", jfbym_token="")
```

| 参数 | 类型 | 说明 |
|---|---|---|
| `entry_url` | str | **必填**。平台入口种子站（如 `https://leyu.com`）。域名动态轮换，只需提供种子站，内部自动解析当前可用域名 |
| `geepass_token` | str | geepass 打码平台 token（纯 HTTP 登录用） |
| `jfbym_token` | str | jfbym 打码平台 token（备用打码渠道） |

### `await client.login(account, password="", force_refresh=False) -> dict`

登录并建立会话。凭证自动缓存（同账号下次调用可省略密码）。

| 参数 | 说明 |
|---|---|
| `account` | 乐鱼账号 |
| `password` | 密码。存在有效缓存时可为空 |
| `force_refresh` | True 时跳过缓存强制走完整登录流程 |

返回：

```python
{
    "account":    str,   # 账号
    "player_id":  int,   # 玩家 ID
    "domain":     str,   # 当前主站域名（动态）
    "game_token": str,   # 游戏会话凭证（敏感，勿外泄）
    "game_exp":   int,   # 凭证过期时间戳（秒）
    "backend":    str,   # 游戏后端地址（动态）
}
```

异常：`LoginError` — 所有登录方式均失败时抛出。

### `await client.get_tables(game_type_id=None) -> list[dict]`

拉取当前大厅桌台列表。需先 `login()`。

| 参数 | 说明 |
|---|---|
| `game_type_id` | 可选。按游戏类型过滤（`2001`=经典百家乐）。`None` 返回全部 |

返回桌台 dict 列表，每项：

```python
{
    "table_id":       int,   # 桌台 ID（进桌用）
    "game_type_id":   int,   # 游戏类型 ID
    "game_type_name": str,   # 游戏类型名（如 "经典百家乐"）
    "table_name":     str,   # 桌台名（大厅快照可能为空，以进桌快照为准）
    "status":         int,   # 桌台状态: 2=下注中 3=发牌中 4=开牌/结算
    "online":         int,   # 在线人数
    "boot_no":        str,   # 靴号
    "road_flat":      str,   # 珠盘路序列 "BBPTPB6..."（B=庄 P=闲 T=和 B6=幸运6庄）
    "road_count":     int,   # 本靴已开局数
}
```

### `await client.enter_table(table_id, game_type_id=2001) -> TableSession`

进入指定桌台，返回 `TableSession`（异步上下文管理器，退出时自动离桌）。
凭证刷新、连接建立、被踢自动重进全部内部处理。

```python
async with await client.enter_table(2656, 2001) as ts:
    ...
```

#### `ts.snapshot: dict`

进桌全量快照，主要字段：

| 字段 | 说明 |
|---|---|
| `tableName` | 桌台名（如 "经典百家乐B10"） |
| `bootNo` | 靴号 |
| `roundNo` | 当前局号 |
| `dealerName` | 荷官名 |
| `cardResult` | 当前牌面结果 |
| `roadPaper` | 路纸原始数据 |
| … | 共 60+ 字段，涵盖桌台配置、限额、状态等 |

#### `ts.road_flat() -> str`

当前珠盘路序列，同 `get_tables` 的 `road_flat` 格式。

### `await client.get_settings() -> list[dict]`

读取当前玩家的全部设置（含路纸筛选偏好）。需先 `login()`。

返回设置项列表，每项：

```python
{
    "playerId":      int,
    "settingType":   str,   # 设置大类，"4" = 大厅筛选
    "settingObject": str,   # 子项: "22"=游戏类型过滤 "23"=路纸类型过滤
    "deviceType":    int,
    "value":         str,   # 选中的 id 列表，逗号分隔
    "defaultValue":  str,   # 默认值
}
```

### `await client.set_setting(setting_object, value, setting_type="4") -> bool`

修改一项玩家设置（持久化到服务端，与官方前端写同一接口）。

| 参数 | 说明 |
|---|---|
| `setting_object` | 子项 id。`"22"`=游戏类型过滤，`"23"`=路纸类型过滤 |
| `value` | 选中的 id 列表，逗号分隔（如 `"2,1,3"`） |
| `setting_type` | 设置大类，默认 `"4"` |

返回 `True` = 写入成功。

### `await client.set_road_filter(road_ids) -> bool`

修改路纸筛选偏好的便捷封装（`set_setting("23", ...)`）。

`road_ids` 取值 1~11，官方名称见 `GOOD_ROAD_NAMES`（从 `hdata.client` 导入）：

| id | 名称 | id | 名称 | id | 名称 |
|---:|-----|---:|-----|---:|-----|
| 1 | 长闲 | 5 | 一庄两闲 | 9 | 逢庄连 |
| 2 | 长庄 | 6 | 一闲两庄 | 10 | 逢闲连 |
| 3 | 大路单跳 | 7 | 逢庄跳 | 11 | 排排连 |
| 4 | 长路转单跳 | 8 | 逢闲跳 | | |

```python
await client.set_road_filter([2, 1])      # 只看长庄+长闲
await client.set_road_filter("2,1,3,5")   # 也接受逗号分隔字符串
```

> 注意：该设置只是把偏好**持久化到服务端**（官方前端据此在本地过滤桌台显示），
> 服务端推送内容不受其影响。用 HData 做监控时通常无需调用——
> `get_tables()` 拿到全量桌台+全长路纸后自行筛选即可。

### `await client.enter_tables(tables, kick_policy="stay") -> MultiTableSession`

同时进入多张桌台监控（**共享一条 WS 连接，一个账号即可，无需多账号**）。

```python
picked = [t for t in await client.get_tables() if 你的筛选(t)]
async with await client.enter_tables(picked) as mts:
    mts.snapshots          # {table_id: 进桌快照}
    mts.road_flat(tid)     # 指定桌当前珠盘路
    async for ev in mts.events():   # 事件含 table_id 标识来源桌
        ...
```

| 参数 | 说明 |
|---|---|
| `tables` | 桌台列表（`get_tables()` 返回项，至少含 `table_id` / `game_type_id`） |
| `kick_policy` | `"stay"`（默认）= 被踢自动重进该桌，监控不中断；<br>`"follow_system"` = 遵循系统踢出，该桌停止监控 |

行为同 `TableSession`：某桌被踢自动重进该桌（不影响其他桌）；退出时自动离全部桌。

### `await client.monitor_tables(tables, accounts=None, kick_policy="stay") -> TableMonitor`

持续多桌监控（**人为主动控制退出，无任何自动超时**），单/多账号自动兼容。

| 参数 | 说明 |
|---|---|
| `tables` | 桌台列表（`get_tables()` 返回项） |
| `accounts` | 可选。额外账号 `[{"account":..,"password":..}, ...]`。<br>不传 = **单账号多桌**（全部桌压当前账号一条连接）；<br>传入 = **多账号多桌**（桌台轮询分配到各账号，每账号一条连接）。当前登录账号自动算第一个，无需重复传 |
| `kick_policy` | 同 `enter_tables`，对每个账号分片生效 |

```python
async with await client.monitor_tables(picked) as mon:
    mon.snapshots              # {table_id: 快照}（跨账号合并）
    mon.road_flat(tid)         # 指定桌当前珠盘路
    mon.table_ids              # 监控中的桌台列表
    await mon.add_table(t)     # 动态加桌
    await mon.leave_table(tid) # 主动退某桌（其他桌不受影响）
    async for ev in mon.events():   # 统一事件流，含 table_id
        ...                         # aclose() 后迭代结束
    # 退出 async with 自动 aclose()（离全部桌、断全部连接）
```

> 设计意图：目前实测单账号一条连接可同时多桌；若平台日后限制，
> 只需在 `accounts` 里补账号即可无缝切换，业务代码不变。

### `road_streak(road) -> (side, count)`

连胜计算（与官方口径对齐）：

- `T`（和）归属于之前最近一局非和局的胜方，**不打断连胜也不计数**
- `B6`（幸运6庄）视为 `B`

```python
>>> from hdata.client import road_streak
>>> road_streak("PTBBB")   # ('B', 3)
>>> road_streak("BTTBB")   # ('B', 3) — 中间2局T视为庄和不占局数
>>> road_streak("BTBBT")   # ('B', 3) — 末尾T归庄
```

#### `async for event in ts.events()`

持续产出桌内事件的异步迭代器。每个事件：

```python
{
    "type":        str,   # 事件分类: round / card / bet / road / status / notice / leave / kick / lobby / other
    "protocol_id": int,   # 原始协议号（102=离桌推送 104=局状态 106/107=发牌 110=桌台动态
                          #                116/160/161=路纸 123=系统通知）
    "table_id":    int,
    "data":        dict,  # 解码后的业务数据（结构随协议号不同）
}
```

行为约定：

- **踢出机制（实测）**：连续未下注满 **3 局**，服务器推送预警（`type="notice"`，协议 123，
  "您已连续3局未下注，后2局未下注将退出游戏"），**只是提醒，不会被踢**；
  满 **5 局**推送离桌通知（协议 102，`leaveTableType=2`，"您长时间未下注，已退出游戏"），
  这才算被踢。**被踢后 WS 连接不断**，只是收不到该桌数据。
  另外：`leaveTableType=1` 是自己主动离桌的确认推送，内部已自动过滤。
- **无法防踢（已实测）**：被踢前重发进桌指令**不能**重置未下注计数，只能被踢后重进。
- **stay（默认）**：被踢后迭代器内部自动重进该桌并产出 `type="kick"` 事件
  （`data.dropped=False`，`data.raw` 为原始离桌推送），随后继续正常产出，调用方无需处理。
- **follow_system**：被踢的桌停止监控，产出 `type="kick"` 事件（`data.dropped=True`）；
  全部桌被踢后迭代结束。
- **会话失效**：凭证被服务端作废时抛出 `LoginError`，需重新 `login()`。
- 退出 `async with` 自动离桌并关闭连接。

## 异常

| 异常 | 触发 |
|---|---|
| `hdata.client.LoginError` | 登录失败 / 凭证失效 / 未登录即调用 |

## 注意事项

1. **域名与端口是动态轮换的**：种子站不变，实际域名/后端可能每小时变化。
   内部已自动解析与探活，调用方无需关心，也不要缓存任何域名。
2. **凭证有时效**：`game_exp` 之后凭证失效。长时间运行建议在捕获 `LoginError` 后重新 `login()`。
3. **打码 token 计费**：`login()` 走纯 HTTP 打码登录会消耗打码额度；有有效缓存时不消耗。
4. **同一账号同时只允许一条游戏连接**：重复 `enter_table` 前请先退出上一个会话。
